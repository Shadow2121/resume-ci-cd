# Rewriting Examples

## Bullet rewriting: weak → strong

**Before (weak):**
> Worked on the backend API for a financial data platform

**After (strong, targeting a data engineering JD):**
> Built RESTful microservices in Python/FastAPI for a financial data platform processing 15K daily transactions, reducing average API response time by 18% through connection pooling

---

## Metric injection examples

| Claim type | Weak | Strong |
|-----------|------|--------|
| Latency improvement | "improved performance" | "reduced p99 latency from 340ms to 190ms" |
| Cost reduction | "optimized cloud costs" | "reduced monthly AWS spend by 23% ($3.2K/mo) via Reserved Instances and S3 lifecycle policies" |
| Reliability | "improved system stability" | "reduced deployment-related incidents by 35% by containerizing services with Docker + Kubernetes rollout strategies" |
| Scale | "handled high traffic" | "served 15,000 daily active users at <200ms p99 latency" |

Metric ranges must stay within 10–25% improvement unless the master already contains a specific number.

## Keyword targeting examples

If JD mentions: `dbt`, `Airflow`, `data warehouse`
And master has: `PostgreSQL, Kafka, Spark`

Valid latent extension (creative buffer):
> Added `dbt` to skills if the candidate works with SQL transformation pipelines (logical extension of existing SQL/Postgres stack)

Invalid fabrication:
> Claiming "built Airflow DAGs for 50-node cluster" with no evidence in the master

## Markdown formatting reference

| Element | Syntax |
|---------|--------|
| Section heading | `## Section Name` |
| Job title | `### Job Title` |
| Company / role line | `**Company Name** \| City \| Dates` |
| Bullet point | `- Bullet text here` |
| Bold (company, skill category) | `**text**` |
| Inline code (tool names) | `` `ToolName` `` — use sparingly |
| Horizontal rule between sections | `---` |

Do not use HTML tags. Do not use `<br>`, `<strong>`, or any other HTML.
Pipe characters `|` in bullet text are fine — no escaping needed in Markdown.
